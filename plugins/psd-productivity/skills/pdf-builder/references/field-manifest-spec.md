# Field Manifest Specification

## Overview

Every PDF generated by `generate_pdf.py` produces a companion `.fields.json` file containing Documenso-ready field coordinates. This manifest maps directly to Documenso's `POST /api/v2/envelope/field/create-many` endpoint.

## Schema

```json
{
  "pdf_path": "/path/to/output.pdf",
  "page_size": {
    "width": 612.0,
    "height": 792.0,
    "unit": "points"
  },
  "pages": 1,
  "fields": [
    {
      "name": "employee_signature",
      "type": "SIGNATURE",
      "fieldMeta": {
        "type": "signature",
        "label": "Employee Signature",
        "required": true
      },
      "page": 1,
      "positionX": 8.82,
      "positionY": 82.45,
      "width": 38.07,
      "height": 5.05
    }
  ]
}
```

## Field Properties

| Property | Type | Description |
|----------|------|-------------|
| `name` | string | Slug identifier (auto-generated from label) |
| `type` | string | UPPERCASE Documenso field category |
| `fieldMeta.type` | string | lowercase Documenso field type |
| `fieldMeta.label` | string | Human-readable label |
| `fieldMeta.required` | boolean | Whether field is required |
| `fieldMeta.placeholder` | string | Optional placeholder text |
| `fieldMeta.options` | array | Options for DROPDOWN fields |
| `page` | integer | 1-based page number |
| `positionX` | float | Left edge, 0-100% of page width |
| `positionY` | float | Top edge, 0-100% of page height |
| `width` | float | Width, 0-100% of page width |
| `height` | float | Height, 0-100% of page height |

## Type Mappings

| Field Type | Outer `type` | Inner `fieldMeta.type` |
|-----------|-------------|----------------------|
| Signature | SIGNATURE | signature |
| Date | DATE | date |
| Text input | TEXT | text |
| Full name | NAME | name |
| Email | EMAIL | email |
| Checkbox | CHECKBOX | checkbox |
| Initials | INITIALS | initials |
| Number | TEXT | number |
| Dropdown | DROPDOWN | dropdown |

**Critical**: The outer `type` is UPPERCASE, the inner `fieldMeta.type` is lowercase. Documenso requires this distinction.

## Using with Documenso

### Interactive (Claude)

```
1. Generate PDF:    /pdf-builder permission-slip
2. Create envelope: /documenso create /tmp/form.pdf
3. Add fields:      Read .fields.json → /documenso add-fields <envelope-id> <manifest>
```

### Programmatic (n8n)

```javascript
// In n8n Code Node — read manifest and build Documenso field payload
const manifest = JSON.parse(fs.readFileSync('/tmp/doc.pdf.fields.json'));
const recipientId = previousNode.json.recipientId;

const fieldPayload = {
  envelopeId: previousNode.json.envelopeId,
  data: manifest.fields.map(f => ({
    recipientId: recipientId,
    type: f.type,
    identifier: 0,  // first PDF in envelope
    page: f.page,
    positionX: f.positionX,
    positionY: f.positionY,
    width: f.width,
    height: f.height,
    fieldMeta: f.fieldMeta,
  })),
};

return [{ json: fieldPayload }];
```
